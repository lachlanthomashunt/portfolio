This is a demonstration of the SQL codes I've generated in MySQL which generates and executes requests from the server.

NOTE: The following code was forward engineered using MySQL and creates resources at the beginning of the script. In most cases, a database, schema and tables would already be created on a server and users/administators would access the database through VIEWS, which limit the destructive privileges users could use on the database.

The code creates a relational database, meaning multiple tables hold data which the users can access and complex requests can return data to the user by referencing multiples tables in a single request. The user must have valid access and privileges to the database in order to carry out these requests. Requests are then sent to the database and returned to the user.

The code executes out the following instructions:

  • Sets some initial parameters,
  • Drops the schema if it exists and creates a schema to use,
  • Uses the database,
  • Drops each table if it exists and creates each table, progressively adding the data types the tables will accept,
  • Populates the tables with data,
  • Generates the result a student scored via their studentID number,
  • Generates the grades that have been scored in a specific subject,
  • Generates the age of a student via their studentID,
  • Generates the highest result from a student,
  • Alters a table to add a column "Spoken Language at home",
  • Alters a table to change the Subject Coordinator of a subject,
  • Demonstrates how Procedures can be declared and then called to execute requests.
